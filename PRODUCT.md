# Product

## Register

product

## Users

Crypto-native developers and power users who live in the terminal and treat PromptFolio as a command-line interface for portfolio operations. They run nodes, read docs, and understand Solana deeply. They open PromptFolio to issue a natural-language command — rebalance, send, query — and expect it to execute correctly and confirm precisely.

## Product Purpose

PromptFolio is an AI-powered Solana portfolio manager that parses natural language into on-chain operations. It eliminates the cognitive overhead of DeFi UIs by letting users describe their intent in plain English. Success looks like: user types a command, the AI parses it correctly, the safety gate confirms the details, and the transaction executes — zero friction, full confidence.

## Brand Personality

Precise · Trustless · Sharp

Voice: Direct and terse. No marketing fluff. Every label earns its place. Confidence without noise.
Tone: Like a Rust compiler output — exact, helpful, and honest about errors.

## Anti-references

- **Generic SaaS dashboard**: No hero-metric cards with gradient stat boxes, no identical card grids with icons, no soft-rounded corners that read as consumer fintech.
- **Neon cyberpunk crypto**: No purple-to-cyan gradients, no glitch effects, no matrix-rain aesthetic, no Degen energy. This is institutional-grade infrastructure, not a nightclub.

## Design Principles

1. **Every pixel earns its place.** No decorative elements. Density is a feature when information is the product.
2. **Precision over personality.** Typography and spacing signal trustworthiness. IBM Plex Mono is the voice; let it speak clearly.
3. **Errors are first-class citizens.** A failed parse or rejected transaction must be as legible as a success. Never bury failures.
4. **The user already knows crypto.** Don't explain what a wallet is. Dense information tables are a feature, not a design problem.
5. **Sharp edges, sharp thinking.** Zero border radius is not a quirk — it signals that this tool is serious infrastructure, not a toy.

## Accessibility & Inclusion

WCAG AA minimum. All body text must meet 4.5:1 contrast ratio against its background. Primary actions must be keyboard-accessible. Reduced-motion media query required on all animated elements.
